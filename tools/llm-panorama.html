1. 全局架构 — 物理 × 逻辑双栏

三模式切换：Prefill / Decode / 训练。

🖥️ 物理硬件

API Gateway / 网络

请求进入、鉴权、路由，网络约 5ms。

CPU 调度 + Tokenizer

连续 batching、队列、tokenize。

PCIe / NVLink

输入传输与多卡通信。

GPU SM + SRAM

片上高速缓存，约 40MB / 19TB/s。

GPU HBM

权重、激活片段、KV Cache，约 80GB / 3.35TB/s。

Host RAM

调度队列、offload、数据加载，约 256GB / 50GB/s。

NVMe SSD

checkpoint、语料、模型文件，TB 级。

🧠 逻辑算法

Prompt → Tokens

字符串变 token id。

Embedding + RoPE

token 变向量，RoPE 注入位置信息。

Attention

QKᵀ / √d → mask → softmax → V。

FFN

逐 token 非线性变换，参数量占大头。

KV Cache

保存历史 K/V，decode 读取而不是重算。

Sampling / Stream

采样并流式返回。

Loss / Backward / Optimizer

训练新增梯度、优化器状态、激活保存。

5. 训练 vs 推理

训练通常不需要 KV Cache，因为训练是全序列 teacher-forcing，而不是逐 token 生成。

推理

低延迟 / 高吞吐 / 低成本

Prefill 并行处理 prompt，Decode 逐 token 读取 KV Cache。只做 forward。

训练

反向传播 / 优化器更新

需要保存激活、梯度、Adam 状态，显存和通信远重于推理。
