01
Nov 2025 Self-published作者自出版 Senior practitioner资深实践者

Designing Multi-Agent Systems

Victor Dibia · Microsoft Research / creator of AutoGen Studio

The single best on-target book in this list. Dibia spent two years writing it after building AutoGen and AutoGen Studio, and he deliberately keeps it framework-agnostic — he ships a teaching framework called PicoAgents built from scratch instead of locking the content to LangGraph or CrewAI APIs that age in months. For someone already running ReAct, Plan-and-Execute, and Supervisor patterns in production, this is the book that gives you the vocabulary and trade-off framing you've been deriving from blog posts. Strong chapters on trajectory-based evaluation, observability, and interruptibility — the things that actually break in production multi-agent systems.

本清单中最切题的一本。作者在 Microsoft Research 主导构建 AutoGen 和 AutoGen Studio 之后，花了整整两年写就此书，并刻意保持 framework-agnostic（与具体框架解耦）—— 他从零实现了一个名为 PicoAgents 的教学框架来讲解原理，而不是把内容绑定到几个月就会过时的 LangGraph 或 CrewAI API 上。如果你已经在生产环境跑过 ReAct、Plan-and-Execute、Supervisor 等模式，这本书能把你从博客文章里零散攒起来的概念，整理成一套清晰的术语体系和权衡框架。trajectory-based evaluation（轨迹评估）、observability（可观测性）、interruptibility（可中断性）几章尤其扎实 —— 这些恰恰是多智能体系统在生产环境中真正会出问题的地方。

Best for适合
Anyone designing supervisor / hierarchical / collaborative agent topologies and tired of framework-specific tutorials. 正在设计 supervisor / 层级式 / 协作式 agent 拓扑、且厌倦了框架特定教程的人。

02
2025 O'Reilly All levels各层级通用

AI Engineering

Chip Huyen · author of Designing ML Systems

The closest thing the field has to a canonical text. Where Dibia goes deep on multi-agent design, Huyen goes wide on the full stack of production LLM systems: evaluation frameworks, prompt design, latency/cost/quality trade-offs, and a serious treatment of building robust evals for non-deterministic, multi-step agents — the hardest unsolved problem in the space. Engineering-first, not research-first. If your team has been arguing about how to actually measure agent quality past "looks right to me," this gives you a defensible vocabulary for those conversations. Pairs naturally with what you've been building for interview rubrics.

目前这个领域最接近"经典教材"的一本。如果说 Dibia 在多智能体架构上做的是"深度挖掘"，Huyen 这本则是"广度覆盖"—— 涵盖生产级 LLM 系统的全栈：评估框架、prompt（提示词）设计、延迟/成本/质量的权衡，以及一段非常硬核的内容：如何为非确定性、多步骤的 agent 构建鲁棒的 evals（评估体系）—— 这是整个领域目前最难、最未解决的问题。视角是"工程优先"而非"研究优先"。如果你的团队还在为"agent 的质量到底怎么衡量"反复争论（除了"看起来对"之外），这本书能给你一套可辩护、可落地的话语体系。和你最近在做的面试评估 rubric（评分量表）工作完美互补。

Best for适合
Engineering leads who need to think about evals, observability, and trade-offs across an entire LLM platform — not just one agent. 需要从整个 LLM 平台层面（而不仅是单个 agent）思考评估、可观测性与权衡的工程负责人。

03
2025 · 2nd ed Packt Hands-on实操

Generative AI with LangChain

Ben Auffarth & Leonid Kuligin · founder of Chelsea AI Ventures

The 2nd edition is essentially a different book — fully rewritten around LangGraph multi-agent architectures and enterprise deployment. The first edition focused on individual developers; this one explicitly addresses engineering teams. Chapters on Tree-of-Thoughts, structured generation, agent handoffs with error handling, and hybrid-search/re-ranking RAG pipelines. If your AgentScope Java work eventually needs a sibling on the Python side — or if you're benchmarking your in-house orchestration against LangGraph patterns — this is the most complete reference.

第二版几乎可视作一本全新的书 —— 围绕 LangGraph 多智能体架构与企业级部署完全重写。第一版面向个人开发者，第二版则明确面向工程团队。涵盖 Tree-of-Thoughts（思维树）、structured generation（结构化输出）、agent 间带错误处理的 handoff（交接）、以及 hybrid-search（混合检索）+ re-ranking（重排序）的高级 RAG 管线。如果你的 AgentScope Java 工作之后需要一个 Python 端的对照实现，或想拿团队自研的编排逻辑去对标 LangGraph 的模式，这本是覆盖最全的参考。

Best for适合
Teams committing to LangGraph for production, or anyone wanting a thorough patterns-and-code companion to Tier 01 + 02. 决定用 LangGraph 上生产的团队，或想要 Tier 01 + 02 的"模式 + 代码"完整对照读物的人。

04
2025 Manning Hands-on实操

AI Agents and Applications: With LangChain, LangGraph and MCP

Roberto Infante · London hedge fund quant

Notable as one of the first practitioner books with an explicit MCP chapter covering exposing, composing, and consuming plug-and-play tools — directly relevant to the shizhuang-data MCP work. Less theoretical than Dibia, more cookbook-shaped: prompt and context engineering, advanced RAG, tool-based agents, multi-agent systems. The FinTech background shows in the chapters on cost/latency budgeting and maintainability. Read alongside Tier 01·02, not after.

目前少数有专门 MCP（Model Context Protocol）章节的实战书 —— 涵盖如何"暴露、组合、消费"即插即用的工具，与你正在做的 shizhuang-data MCP Server 直接相关。理论性不如 Dibia 那本，更偏 cookbook（菜谱）风格：prompt 与 context engineering（上下文工程）、高级 RAG、工具型 agent、多智能体系统。作者的 FinTech 背景体现在成本/延迟预算与可维护性几章上。建议与 Tier 01·02 并行阅读，而不是读完它们之后再看。

Best for适合
MCP server builders, anyone who wants concrete code for tool orchestration with branching logic. MCP server 构建者，以及想要"带条件分支的工具编排"具体代码的人。

2025 O'Reilly Reference工具书

Generative AI Design Patterns

Valliappa Lakshmanan · author of ML Design Patterns

Thirty-two named patterns covering RAG variants, reasoning strategies, tool use, and agent topologies — written in the same Gang-of-Four style as his earlier ML Design Patterns book. Not a tutorial. Treat it as the reference shelf you reach for when you've named a problem and want to see the catalog of solutions. Particularly useful when reviewing an architecture proposal: "is this Pattern X with the trade-offs we talked about, or are we inventing one?"

32 个命名模式，覆盖 RAG 的各种变体、reasoning（推理）策略、工具调用、agent 拓扑结构 —— 沿用作者上一本《ML Design Patterns》的 Gang-of-Four（设计模式四人帮）写作风格。不是 tutorial，而是工具书 —— 当你已经能命名一个问题、想查阅可选方案目录时翻开它。在做架构评审时尤其好用："这是不是我们之前讨论过的 X 模式（连同它的权衡）？还是我们在重新发明轮子？"

Best for适合
Architecture reviews, design docs, and getting unstuck when you suspect there's a known pattern you're missing. 架构评审、设计文档、以及当你怀疑"漏掉了某个已知模式"时的解药。

06
Late 2024 Packt Engineering ops工程运维

LLM Engineer's Handbook

Paul Iusztin & Maxime Labonne · Decoding ML

The most explicit "how do you actually run this in production" book on the list — fine-tuning, deployment, monitoring, data pipelines for LLM apps, and an end-to-end project. Less agent-centric than the others, more concerned with the surrounding MLOps/LLMOps machinery that lets agents exist reliably. Useful counterweight: a lot of agent writing assumes the model layer is solved; this book is the reminder that it isn't.

清单中最直接回答"这东西到底怎么跑在生产环境"的一本 —— fine-tuning（微调）、部署、监控、LLM 应用的数据管线、以及一个端到端项目。Agent 视角比其他几本少，更关注让 agent 能可靠运行所必需的 MLOps / LLMOps 基础设施。一个有用的"反向制衡"：大多数 agent 类书籍假设模型层的问题已经解决，这本书提醒你 —— 并没有。

Best for适合
Platform engineers responsible for the model-serving and data-pipeline layers under an agent system. 负责 agent 系统底层模型服务与数据管线的平台工程师。

Skipped on purpose刻意未选

and why及原因

Build a Large Language Model (From Scratch) — Sebastian Raschka

Excellent book, wrong altitude for this list. Worth reading if you want to deepen LLM internals, but as a senior agent engineer your bottleneck is almost certainly orchestration and evaluation, not transformer math.

非常好的书，但不在这个清单的"高度"上。如果想深入理解 LLM 内部机制，值得一读；但作为资深 agent 工程师，你的瓶颈几乎一定在编排与评估，而不是 transformer 数学。

Building LLMs for Production — Bouchard & Peters

Solid entry-to-mid-level book, but substantially overlapped by Huyen and Iusztin. Pick those instead unless you specifically want the Towards AI house perspective.

中等难度、扎实，但内容与 Huyen 和 Iusztin 那两本大幅重合。除非你特别想看 Towards AI 团队的视角，否则优先选那两本。

Any framework-specific deep dive (CrewAI, AutoGen, etc.)任何框架特定的深度专著（CrewAI、AutoGen 等）

Their APIs shift faster than print cycles. Read the docs and the source. Dibia explicitly excluded these from his own recommendations for the same reason.

这类框架的 API 变化速度远快于纸书出版周期。直接读文档和源码。Dibia 自己也基于同样原因，把这类书从他的推荐里排除了。
